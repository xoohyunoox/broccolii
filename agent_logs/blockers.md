# Blockers

Hard blockers — work that cannot proceed on **any** track without resolution. Most issues should go in `questions_queue.md` with a provisional assumption instead; this file is only for true dead-ends.

If something lands here, the orchestrator should also end the session early and surface it in the summary.

---

## Template (for each blocker)

```
### {short title}

**Logged:** YYYY-MM-DD HH:MM

**What's blocked:** {feature / track / file}

**Why:** {root cause}

**What's needed to unblock:** {specific action, ideally with owner}

**Workarounds attempted:** {what was tried before declaring blocker}
```
