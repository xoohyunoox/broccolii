# 브로콜리(Broccolii) 한국 헬스앱 — API·법률 리서치 보고서

## TL;DR
- **API**: 개인 개발자가 실제로 쓸 수 있는 옵션은 사실상 **틸코(Tilko)뿐**이다. 마이헬스웨이/건강정보고속도로는 사업자등록·지정심사·보안취약점 점검 결과서 제출이 필수라 자연인 신청은 불가하고, 공공데이터포털 NHIS·HIRA OpenAPI는 통계 가공·기관 정보뿐이라 "내 검진결과" 조회는 불가능하다. 정부 공식 PHR인 **나의건강기록 앱**(보건복지부·K-HIS)이 이미 자녀(주민등록상 동거 14세 미만)·예방접종·진료·검진 통합 조회를 무료로 제공하므로, 브로콜리는 분석·시각화·가족공유의 차별화가 필요하다.
- **아동(미성년) 데이터**: 친권자(부모)는 의료법 제21조·시행규칙 제13조의3·개정 개인정보보호법 제22조의2상 자녀 데이터에 대한 동의·열람권을 행사할 수 있으나, **시나리오 B(자녀가 다른 보호자의 건강보험 피부양자)**는 NHIS 본인인증이 다른 보호자 명의로만 가능하므로 (a) 다른 보호자 휴대폰 인증 위임(법적 회색지대) 또는 (b) 의료기관·NHIS 지사 오프라인 발급 후 PDF 업로드(가장 안전) 두 가지 워크어라운드만 현실적이다.
- **치매가족 데이터**: 공식 후견인 지정이 없으면 의료법 제21조 제3항의 "환자 동의" 또는 "환자 본인 동의 불가 + 친족 증빙" 요건을 충족해야 한다. **in-app 디지털 동의로 의료법 시행규칙의 자필서명 요건을 완전 대체하는 것은 법적으로 안전하지 않다.** "프록시 프로필"은 자필 동의서 사진 + 가족관계증명서·진단서 업로드 + 명시적 면책 동의 UI로 설계해야 하며, 자동 스크래핑은 본인 프로필에만 허용해야 한다.

## Key Findings

1. **틸코(Tilko)는 이메일·닉네임·비밀번호만으로 가입 가능**(사업자등록증 불요)하며, 건당 과금 모델(NHIS 건강검진내역 300포인트/건, 영유아 건강검진 100p, 진료·투약 정보 150p, HIRA "내가 먹는 약" 300p). 신규 가입시 10,000포인트 무료 제공, 월정 의무 없음. 그러나 틸코 개인정보처리방침 §5의 위탁자 표가 비어 있어 개발자가 PIPA 제26조 처리위탁 계약을 별도 체결해야 책임관계가 명확해진다.
2. **인증**: 틸코는 [공동인증서용]·[간편인증용] 두 트랙. 카카오·통신사 PASS·삼성패스·KB·신한·네이버·토스 등 NHIS·HIRA·정부24 표준 간편인증 사업자가 지원된다.
3. **건강정보고속도로(마이헬스웨이)** = 보건복지부·한국보건의료정보원 운영. 활용기관 회원가입 단계부터 사업자등록번호 필수, **지정심사 → 테스트 정보이용 신청 승인 → 운영전환 신청(보안취약점 점검 결과서·테스트 결과서 첨부 필수)** 단계를 통과해야 함. 개인은 신청 자체 불가.
4. **공공데이터포털 NHIS·HIRA API**는 "건강검진정보(100만명 표본)" 같은 연구·통계용 가공 데이터셋과 검진기관·병원정보 API뿐, 개인 본인 데이터 조회 API는 없다.
5. **나의건강기록 앱**(보건복지부 공식): 보건복지부·K-HIS 2024.9.26 보도자료에 따르면 현재 **1,004개 의료기관**이 연계되어 있으며 "2차 확산사업이 완료되는 2025년 하반기부터는 전체 상급종합병원 47개소를 포함한 총 1,263개소로 확대될 것으로 예상된다." 같은 보도자료에서 "14세 미만 자녀 건강기록열람은 '나의건강기록' 앱에서 14세 미만의 자녀를 등록하여 자녀의 의료정보를 확인할 수 있는 서비스이다. 주민등록상 부모와 동일한 거주지에 있는 자녀 정보를 '나의건강기록' 앱에 등록하면, 자녀의 건강기록을 확인할 수 있다"고 명시. 2025년 개편으로 대리인 등록 기준이 **만 19세 미만**으로 확대됐고, 향후 치매환자·발달장애인 대리인 등록 확대 예정이다.
6. **CODEF**(developer.codef.io) — 또 다른 민간 스크래핑 API. "건강iN 건강검진결과" 등 제공. 가격은 영업 견적.
7. **개인정보보호법 핵심 트리거**(2025.10.2 시행 시행령 제35780호 기준): **5만명 이상 민감정보·고유식별정보 처리시** 이용·제공내역 통지 의무(법 제20조의2). **손해배상책임 보장 의무**(법 제39조의7, 시행령 제48조의7)는 ① 전년도 매출액 10억원 이상, ② 전년도 말 기준 정보주체수 1만명 이상 두 조건을 모두 충족할 때 발생(2025.9.23 개정으로 기존 매출 5천만/이용자 1천명에서 상향). 위반시 과태료 2천만원.
8. **충돌시 우선순위**: 개인정보보호법 제6조 — 다른 법률의 특별규정 우선. 진료기록 관련은 의료법 제21조·제22조가 특별법으로 우선(법제처·보건복지부 일관된 입장).

## Details

### AREA 1 — APIs

#### A. 틸코(Tilko / 틸코블렛, tilko.net)
(주)틸코블렛이 운영하는 공공·금융기관 스크래핑 API SaaS. 회사소개 페이지에 따르면 "복잡하고 어려운 공인인증, 전자인증 및 암복호화 모듈들이 필요한 각종 공공기관 및 금융기관 등의 스크래핑 솔루션을 간단하고 쉬운 API 형태로 제공"한다. AES-CBC-128 + RSA 공개키 암호화 강제, REST POST 방식.

**제공 API(건강 카테고리)**
- NHIS 건강검진내역(최근 10년치 일반·암 검진)
- NHIS 영유아 건강검진 결과
- NHIS 진료 및 투약 정보(2022.1월 신규 추가)
- NHIS 건강보험자격득실내역
- NHIS 병원 내원 이력 / 약국 이용 이력
- HIRA "내가 먹는 약"
- NHIS 맞춤형 건강정보 / 건강보험료 납부내역
- 예방접종확인서/내역(블로그에 마케팅 — 공식 가격표엔 별도 항목 미공개)

**가격(2023.12.1 개정 단가)** — 틸코 공지 https://tilko.net/Support/Detail/90 ("서비스 제공을 위한 기술적 인프라 확장, 물가 상승에 따른 서버 유지 및 운영 비용의 상승") 및 각 API 데모 페이지에서 직접 확인됨:
| API | 건당 포인트(=원) |
|---|---|
| NHIS 건강검진내역 | 300 (데모 페이지 Ggpab003M0105: "포인트: 300 / 트랜잭션(건)") |
| NHIS 영유아 건강검진 | 100 |
| NHIS 진료 및 투약 정보 | 150 (기존 100 → 인상) |
| HIRA 내가 먹는 약 | 300 |
| 간편인증 완료여부 확인 | 20 |
| HIRA 간편인증 요청 | 100 |

- 50만원 충전시 5%, 500만원 충전시 10% 추가 적립.
- 최저 사용료/월정 의무 없음(prepay only). 신규 가입시 10,000포인트 무료.

**인증 방식**: [공동인증서용] 트랙(CertFile/KeyFile/CertPassword를 AES 암호화) 또는 [간편인증용] 트랙(카카오·통신사 PASS·삼성PASS·KB·신한·네이버·토스 등 민간 간편인증). 간편인증은 "간편인증 요청 → 사용자가 모바일 앱에서 인증 → 간편인증 완료여부 확인 → 실제 데이터 API"의 2-step.

**개인 개발자 가입 가능 여부**: 가능. 회원가입은 이메일·닉네임·비밀번호만 필수(틸코 이용약관 제7조 제2항). 다만 정보주체(end-user) 동의 수집 책임은 약관에 명시되지 않아 개발자가 부담. PIPA 제26조 처리위탁 계약을 별도 체결할 것을 권고.

#### B. 건강정보고속도로(마이헬스웨이, myhealthway.go.kr)
- **운영**: 보건복지부(주관) + 한국보건의료정보원(전담운영기관) + 한국지능정보사회진흥원.
- **제공 데이터**: 공공기관(NHIS·HIRA·질병관리청) + 1,004개 의료기관(2025년 하반기 1,263개소·47개 상급종합병원 전부 연계 목표). 틸코로는 도달할 수 없는 의료기관 1차 진료기록·진단검사·수술내역까지 접근 가능.
- **활용기관 심사 절차**: 기관회원가입(사업자등록번호 필수) → 지정심사 → 테스트 정보이용 신청 → 운영전환 신청(테스트 결과서·보완취약점 점검 결과서 첨부 필수) → 운영 승인. 마이헬스웨이 포털: "승인요청 시 서비스 검증을 위해서 테스트 결과서와 보완취약점 점검 결과서를 반드시 첨부해야만 합니다."
- **개인 개발자 신청 가능 여부**: 불가. 기관회원가입 단계부터 사업자등록번호로 기관 검색이 필요.

#### C. 건강보험공단(NHIS) 직접 API
- NHIS는 외부 개발자용 "내 건강검진 결과" 조회 API를 공개하지 않음. 본인은 NHIS 홈페이지 "건강iN" 또는 The건강보험25시 앱에서 본인인증 후 열람.
- 공공데이터포털에 NHIS가 등록한 OpenAPI는 검진기관별 정보, 100만명 표본 통계 데이터셋 등에 한정.

#### D. 질병관리청 예방접종도우미(nip.kdca.go.kr)
- 예방접종도우미 누리집에서 본인인증 후 본인·자녀(보호자 인증) 접종 이력 조회 가능.
- 공개 OpenAPI는 위탁의료기관 현황 등 기관 정보에 한정. 개인 접종이력 API는 비공개.
- 개인 예방접종 이력은 **나의건강기록 앱 / 건강정보고속도로 / 틸코 스크래핑**을 통해서만 사실상 접근 가능.

#### E. 공공데이터포털(data.go.kr) 헬스 OpenAPI
- 개인이 무료 신청 가능한 헬스 OpenAPI는 모두 통계·기관 정보:
  - 국민건강보험공단_건강검진정보(2024.12.31 기준 100만명 표본)
  - 국민건강보험공단_검진기관별 정보제공
  - 건강보험심사평가원_병원정보서비스 / 의료기관별상세정보서비스
  - 질병관리청_어린이 국가예방접종 위탁의료기관 현황
- 개인 본인 검진 결과·예방접종·진료 내역을 가져올 수 있는 OpenAPI는 없음.

#### F. 기타
- **CODEF** (developer.codef.io): 민간 스크래핑 API.
- **바로써트** (barocert.com): 카카오·네이버·토스·PASS 등 다수 간편인증 사업자 통합 연동.
- **나의건강기록 앱**: 정부 공식 PHR(공공기관 + 1,004개 의료기관 연계). 무료. 14세 미만 자녀·19세 미만 자녀 대리인 등록 지원. 브로콜리의 직접적인 비교 기준.

### AREA 2 — 아동 건강데이터 법률

#### 1. 개인정보보호법 — 14세 미만 아동
- 건강·의료정보는 **민감정보**(법 제23조). 일반 동의와 별도 동의 필요.
- 14세 미만 아동: **법정대리인(친권자) 동의 필수**(법 제22조의2). 위반시 5년 이하 징역 또는 5천만원 이하 벌금, 전체 매출 3% 이내 과징금.
- 법정대리인 동의 확인 방법(시행령 제17조의2): 인터넷 페이지 동의 표시 + 부모 휴대폰 SMS 알림 / 부모 휴대폰 본인인증 / 신용카드 인증 / 전자우편·통화 / 신분증 사본 등.
- 동의 확인용 부모 정보는 동의 확인 후 5일 이내 파기(표준 개인정보 보호지침 제13조). 단 분쟁 대응 입증용 최소정보(동의서·이메일·녹취)는 회원 탈퇴까지 보관 가능(개인정보위 김·장 해설).

#### 2. 의료법 — 미성년자 진료기록 열람
- 의료법 제21조 제2항: 환자가 아닌 다른 사람에게 환자에 관한 기록을 열람하게 해선 안 됨.
- 시행규칙 제13조의3 제2항: 14세 미만은 친권자(법정대리인)가 자필서명 동의서를 작성. 17세 이상이고 의사능력이 있으면 환자 본인 자필서명 필요.
- 자녀 진료기록 사본 발급에 필요한 서류: 친권자 신분증, 가족관계증명서(3개월 이내), 자필서명 동의서/위임장, 자녀 신분증 사본(가능한 경우).

#### 3. 건강보험법 — 시나리오 A vs B
- **시나리오 A (자녀가 본인 피부양자)**: NHIS 건강iN에서 가입자 본인 인증 후 "자녀(영유아) 건강검진 정보" 메뉴 → 검진대상 조회, 문진표 작성, 결과 조회 가능. 영유아 건강검진표는 직장가입자·세대주에게 전자문서/우편으로 발송. NHIS 시스템 자체가 가입자→피부양자 조회 허용 구조.
- **시나리오 B (자녀가 다른 보호자의 피부양자)**: NHIS는 가입자 본인의 인증으로만 본인+피부양자 조회를 허용 → 부모가 가입자가 아니면 데이터에 접근 못함. 의료법상 친권이 있어도 NHIS 데이터에는 가입자 인증이 별도 장벽.
  - **솔루션 1**: 다른 보호자(예: 조부모, 양육권 보호자)의 휴대폰을 일시 사용해 간편인증 → 자녀 데이터 다운로드 → 부모 앱에 PDF/이미지 이관. 법적 회색지대(타인 명의 인증).
  - **솔루션 2 (가장 안전)**: 의료기관·NHIS 지사에서 친권자 권한으로 사본 발급 → 앱에 PDF 업로드.
  - **솔루션 3 (향후)**: 마이헬스웨이 대리인 등록 정책이 이혼·별거가정 친권자까지 확대될 가능성에 대비해 모니터링.

#### 4. 앱이 갖춰야 할 동의 화면(아동)
- 가입시 만 14세 미만 여부 체크 → 미만이면 법정대리인 인증 플로우 강제.
- 분리된 동의: (i) 일반 개인정보 수집·이용, (ii) **민감정보(건강정보) 별도 동의**, (iii) 제3자(틸코·마이헬스웨이 등) 제공 동의, (iv) 처리위탁 동의.
- 14세 미만에게는 명확하고 알기 쉬운 언어 사용(제22조의2 제3항).
- PIPC 「아동·청소년 개인정보 보호 가이드라인」(2024) 권고: 기본설정 "높은 수준", 위치추적 기본 비활성, 동의 확인 후 부모정보 신속 파기, 부정사용 방지 장치.

### AREA 3 — 치매환자 가족 데이터 법률

#### 1. 의료법 제21조 정밀 분석
- 제3항 제1호: 환자의 배우자, 직계 존속·비속, 형제·자매(배우자·직계 존비속 부재시), 배우자의 직계 존속이 **환자 본인의 자필서명 동의서 + 친족관계증명서 + 신분증 사본**으로 요청.
- 제3항 제2호: 환자가 지정한 대리인이 환자 본인 동의서 + 위임장 + 대리권 증명 서류로 요청.
- 제3항 제3호: **환자가 사망/의식불명/중증질환으로 동의를 받을 수 없는 경우** 친족이 친족관계증명서 + 환자 동의 불가 사유 증빙(사망진단서, 의식불명/중증질환 증명서)으로 요청.
- **진료내역 vs 건강검진내역 구분**: 의료법은 "진료기록" 전반에 적용되며 검진기관도 의료기관. NHIS가 보유한 건강검진 결과는 NHIS가 정보주체로부터 동의받아 제공하는 것으로, NHIS 본인인증 절차가 동의 기능을 대신함.
- 보건복지부 의료기관정책과 정책 제2023-322호 "대리인의 진료기록 열람 및 사본발급 관련 안내"(2023.7): 일부 의료기관의 부당한 거부·지연 사례를 지적, 위반시 시정명령(제63조), 미이행시 의료업 정지·개설허가 취소(제64조).

#### 2. 개인정보보호법 v. 의료법
- 개인정보보호법 제6조(다른 법률과의 관계) — 다른 법률의 특별규정 우선. 의료법 제21조·제22조의 진료기록 관련은 특별법으로 우선. 법제처·보건복지부·법원 일관된 입장.
- "정당한 이익" 예외는 민감정보에 매우 제한적 적용. 건강정보는 별도 동의 원칙.

#### 3. 성년후견제도(민법 제9조~제14조의3)
- **성년후견**: 사무처리능력 지속 결여(중증 치매). 가정법원 심판, 정신감정 필수(가사소송법 제45조의2), 2~3개월 소요. 청구권자: 본인, 배우자, 4촌 이내 친족, 미성년/한정/특정후견인·감독인, 검사·지자체장.
- **한정후견**: 사무처리능력 부족(경증~중등도). 부분적 권한.
- **특정후견**: 일시적·특정한 사무(예: 의료기록 열람·관리)에 한정. 감정 대신 의사·전문가 의견 청취. 가장 빠르고 가벼움 — **치매 가족이 의료데이터 열람만 필요한 경우 현실적 선택지**.
- **임의후견**: 본인이 의사능력 있을 때 공정증서로 미리 후견계약 체결 → 등기.

#### 4. Case A — 중증 치매 (동의 불가)
- **솔루션 1**: 시행규칙 제13조의3 제3항 — 친족이 가족관계증명서 + 환자 의식불명/중증질환 입증서류(진단서, 입원확인서)로 의료기관에 사본 발급 요청 → 앱에 PDF 업로드.
- **솔루션 2**: 가정법원에 **특정후견 심판 청구**(의료정보 열람·관리 권한으로 한정) → 후견등기사항증명서를 앱에 업로드 → 후견인이 디지털 동의 행사.
- **솔루션 3 (앱 설계)**: "프록시 프로필"은 자동 데이터 스크래핑이 아니라 가족이 받아온 PDF/이미지를 업로드·정리해 주는 도구로 설계 → 의료법 제21조 위반 리스크 회피.

#### 5. Case B — 경증 치매 (부분 동의 가능)
- **솔루션 1**: 환자 본인이 자필서명 동의서·위임장 작성(시행규칙 제13조의3 제1호) → 가족이 사본 발급 → 앱 업로드. 위임 시점에 환자 의사능력 추정 가능해야 함.
- **솔루션 2 (가장 안전)**: 환자 본인을 회원 가입시켜 환자가 직접 in-app에서 간편인증·동의 → 가족 계정에 "공유" 권한 부여. 환자가 인증을 직접 수행하므로 의료법 우회 없음.
- **법적 요건**: in-app 디지털 위임만으로 시행규칙의 "자필서명" 요건을 대체할 수 있는지 명확한 판례·유권해석 부재 → 보수적으로 자필서명 동의서 사진 업로드 + 환자 신분증 사본 업로드 병용 권장.

#### 6. 앱 적용 — 법적 리스크와 안전한 설계
- **개인 개발자의 형사·행정 리스크**: 정보주체 동의 없이 민감정보 처리시 5년 이하 징역 또는 5천만원 이하 벌금(법 제71조), 매출 3% 과징금(제64조의2). 의료법 위반은 의료기관 처분이 원칙이나, 부정 발급 자료를 받아 처리한 경우 공범·교사 책임 가능.
- **5만명 이상 민감정보·고유식별정보 처리 트리거**(법 제20조의2): 이용·제공내역 매년 정보주체 통지 의무.
- **손해배상책임 보장 의무**(법 제39조의7, 시행령 제48조의7, 2025.10.2 시행): ① 전년도 매출 10억원 이상, ② 전년도 말 기준 정보주체수 1만명 이상 — **두 조건 모두 충족시** 보험·공제 가입 또는 준비금 적립 의무. 미이행시 과태료 2천만원. (2025.9.23 시행령 개정으로 기존 매출 5천만원/이용자 1천명에서 상향)
- **권장 디스클레이머/동의 화면**:
  1. "본 앱은 의료법 제21조의 적법한 자료 제공 절차를 통해 발급된 자료만을 처리하며, 무권 열람·발급된 자료의 업로드는 금지됩니다."
  2. "타인(가족) 데이터 등록시 본인이 의료법상 적법한 권한자임을 보증하며, 위반시 모든 책임은 사용자가 부담합니다."
  3. "치매·중증질환 환자 데이터 처리시 친족관계증명서·환자 동의 불가 사유 증빙·진단서를 사전 업로드해야 합니다."
- **"안전한" 프록시 프로필 설계**: 자동 스크래핑(틸코 호출) 없이 PDF·이미지·수기 입력만 받아 정리·시각화하는 모드를 기본값으로. 자동 스크래핑은 본인 프로필에만 활성화.

### 추가 — 가이드라인·판례·개인 개발자 의무

- **PIPC 가이드라인**: 「보건의료 데이터 활용 가이드라인」 — 보건복지부 보도자료 및 K-HIS 공지에 따르면 2024.12.16 개정(3차 개정). 개정 이력: 2020.9 최초(개인정보위·복지부 공동) → 2022.1 → 2024.1 → **2024.12.16(AI 비정형 의료데이터 가명처리 절차 구체화 중점, 복지부 단독 개정)**. 주로 **가명정보 활용**(연구·통계)에 대한 것이지만, 보안 조치·DRB 심의·민감정보 결합 기준은 참고가치 큼.
- 「아동·청소년 개인정보 보호 가이드라인」(2024): 14세 미만 UX 설계 권고(연령 직접 입력, 기본설정 "높은 수준", 위치추적 기본 비활성 등).
- **판례·결정례**: 위 보건복지부 정책 제2023-322호 외에는 헬스앱 프록시 데이터 접근에 관한 공개 PIPC 결정례·법원 판결이 많지 않음 → 회색지대 잔존.

**개인 개발자가 민감정보를 처리할 때 의무**:
1. 개인정보처리방침 수립·공개(제30조).
2. 개인정보보호책임자(CPO) 지정(제31조) — 개인 개발자도 본인이 CPO 가능.
3. 안전성 확보조치(제29조 + 시행령 제30조 + 고시): 접근통제, 암호화(주민번호·고유식별정보·바이오정보 강제 암호화), 접속기록 1년 보관(민감정보는 2년).
4. **5만명 이상 민감정보·고유식별정보 처리시** 이용·제공내역 매년 통지(제20조의2).
5. **매출 10억원 + 정보주체 1만명** 두 조건 충족시 손해배상책임보험·공제 의무.
6. 민감정보 별도 동의(제23조).
7. 유출 통지·신고(제34조): 정보주체에 지체 없이 통지, 1천명 이상 유출시 PIPC·KISA 신고.
8. 국외이전 별도 동의 또는 표준계약(제28조의8) — AWS US 등 해외 서버 이용시 필수.
9. ISMS-P 인증은 매출 100억원·일평균 100만명 이상에서 의무.

## Recommendations

### 단계 1 — MVP (1~3개월)
- **본인 데이터만 제공**으로 출발. 틸코 [간편인증용] API로 NHIS 건강검진·진료·투약 + HIRA "내가 먹는 약" 연동(건당 100~300원).
- 가족·아동·치매 프록시는 **PDF/이미지 업로드 + 수기 입력**만 제공. 자동 스크래핑은 본인 한정.
- 만 14세 미만 가입 차단 또는 보호자 휴대폰 인증 강제.
- 손해배상책임보험은 임계값(매출 10억·정보주체 1만명) 이전이라도 출시 전 임의 가입 권장(연 30~70만원대).

### 단계 2 — 자녀(시나리오 A) 자동 연동 (3~6개월)
- 부모(가입자) 인증으로 NHIS·예방접종도우미 자녀 데이터 자동 연동. 영유아 건강검진 100원/건으로 비용 부담 낮음.
- 자녀 등록 시 가족관계증명서 업로드 + 친권자 자기선언 + 휴대폰 본인인증 3중 확인.
- 시나리오 B는 베타로 두고, "다른 보호자 인증 위임" UI는 명시적 면책 + 다른 보호자 휴대폰 본인인증 2단계 요구.

### 단계 3 — 치매 프록시 (6~12개월)
- 데이터 수기·업로드 모드로 출발. 가족관계증명서·진단서·자필서명 동의서 업로드 강제.
- Case B(경증)는 환자 본인 in-app 가입·인증 + 가족계정 공유 패턴 우선.
- Case A(중증)는 특정후견 등기증명서 있으면 정식 위임 처리. 없는 경우는 오프라인 발급 자료 업로드만 허용.

### 단계 4 — 마이헬스웨이 활용기관 신청 (12개월~)
- 사용자 1,000명·매출 의미있는 규모 도달시 1인 사업자 등록 → 마이헬스웨이 활용기관 가입 → 지정심사 신청.
- 마이헬스웨이 연계시 의료기관 1차 진료기록까지 접근 → 틸코 의존도 감소, 정부 공식 채널로 신뢰도 상승.

### 임계값 (트리거 → 대응)
- **민감정보 정보주체 5만명**: 이용·제공내역 매년 통지 시스템 구축(제20조의2).
- **매출 10억원 + 정보주체 1만명**: 손해배상책임보험·공제 의무 가입.
- **민감정보 10만명**: 외부 변호사·CISO 자문 정기화, 사업자 전환 권고.
- **사용자 100만명 또는 매출 100억원**: ISMS-P 인증 의무.

## Caveats

1. **틸코의 예방접종 내역·학생 건강검진** 단가는 공개 가격표(2023.12.1 공지)에 명시되지 않음 — 영업 견적 필요(help@tilko.net). 영유아 건강검진 API 또는 별도 계약에 포함될 가능성.
2. **틸코 약관**은 정보주체(end-user) 동의 수집 책임을 명시하지 않음. 개발자가 자체 동의 UI + PIPA 제26조 처리위탁 계약을 별도 체결해야 안전.
3. **시나리오 B의 "다른 보호자 휴대폰 인증 위임"**은 전자서명법·NHIS 약관상 회색지대. PIPC 결정례 부족 — 규제 리스크 잔존.
4. **in-app 디지털 동의**가 의료법 시행규칙의 "자필서명" 요건을 대체하는지에 대한 명확한 판례·유권해석 부재. 보수적으로 자필서명 사진 업로드 + 디지털 동의 병용.
5. **마이헬스웨이 대리인 등록 정책**이 향후 치매환자·발달장애인까지 확대 예정(K-HIS 공식 발표) → 정책 변동 모니터링 필요.
6. **나의건강기록 앱**(정부 공식, 무료)이 자녀·고령자 대리 기능을 빠르게 확대 중 → 브로콜리는 분석·시각화·가족 공유·UX 차별화가 핵심.
7. 개인정보 보호법 시행령은 2025.9.23 개정(2025.10.2 시행)으로 손해배상책임보장 임계값이 상향됐고, 보건의료 데이터 활용 가이드라인도 2024.12.16 개정 — **본 보고서는 2026년 5월 기준이며, 출시 전 변호사 최신 자문 권장**.
8. 1인 자연인이 NHIS·HIRA 민감정보 수만 건을 처리하는 모델은 PIPC 감독 대상이 될 가능성이 큼. 사용자 1만 명 도달 전에 사업자 등록·CPO 지정·ISMS(P) 자체 점검을 마치는 것을 권고한다.
